# Migrate Tenant DB from Percona to Postgres

## Objective

Consolidate our database stack by removing Percona (MySQL) entirely and exclusively using PostgreSQL for both Application Data (users, auth, tokens) and Tenant Data (medical KPIs, patients, visits, etc.). This simplifies local development, infrastructure, testing, and the AI context.

## Key Changes

- Convert the legacy MySQL dump into standard PostgreSQL DDL & DML.
- Strip out MySQL drivers (`mysql2`) and replace with unified PG queries.
- Ensure LLM generates `postgres` compliant SQL.
